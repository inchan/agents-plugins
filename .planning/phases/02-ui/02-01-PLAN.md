Phase 1 스텁(51줄)을 완전한 에이전트 정의로 교체한다. YAML frontmatter는 유지하되 tools 필드에 MCP 도구 참조를 추가한다. 본문은 다음 5개 섹션으로 구성:

**INIT: 도구 감지 (1회 실행, 캐싱)**
- mcp__playwright__navigate 호출 시도 → 성공 시 available_tools["playwright"] = true
- mcp__chrome-devtools__navigate_page 호출 시도 → 성공 시 available_tools["chrome_devtools"] = true
- Glob으로 agent-browser 스킬 존재 확인 → 발견 시 available_tools["agent_browser"] = true
- primary_tool = 첫 번째 true인 도구; 모두 false면 "code-analysis"
- 각 실패 시 `[WARN ] [BROWSER] +<time> -- Degraded mode: <tool> unavailable` 로그 (browser-automation.md의 Degradation Chain 형식 그대로)

**EXTRACT: 티켓 정보 추출**
- URL 추출: browser-automation.md의 4단계 우선순위 그대로 (Explicit > localhost > Steps > Any URL)
- Viewport 추론: browser-automation.md의 5가지 신호 테이블 그대로 (기본값: 1280x720)
- Reproduction steps 파싱: 11가지 Step Types 테이블 + 트리거 키워드 매핑

**EXECUTE: 재현 단계 실행**
- primary_tool에 따라 도구별 매핑 실행:
  - Playwright: mcp__playwright__navigate, click, fill, evaluate, screenshot 등
  - Chrome DevTools: mcp__chrome-devtools__navigate_page, click, evaluate_script, take_screenshot 등
  - agent-browser: 자연어 지시문으로 전달
- 각 step 실행 실패 시:
  - E_NOTFOUND: 셀렉터 대안 시도 (Shadow DOM이면 evaluate로 직접 접근)
  - E_TIMEOUT: 1회 재시도 후 스킵
  - E_TOOL: 폴백 체인의 다음 도구로 전환
- Shadow DOM 폴백: Playwright click/fill 실패 시 evaluate로 `document.querySelector('host').shadowRoot.querySelector('inner')` 직접 접근

**CAPTURE: Before 스크린샷**
- 전체 페이지: /tmp/wf_{ticket_id}/before_full.png (fullPage: true, waitForLoadState: "networkidle")
- 영향받는 요소: /tmp/wf_{ticket_id}/before_element.png (selector: affected_element)
- Bash로 /tmp/wf_{ticket_id}/ 디렉토리 생성
- 스크린샷 실패 시 → 다음 폴백 도구로 재시도, 최종적으로 "NO_SCREENSHOT" 상태 기록

**OUTPUT: 구조화된 결과**
- browser-automation.md의 "Into Evidence Report" JSON 구조 그대로 출력
- reproduction_status: success | partial | failed
- method: browser:playwright_mcp | browser:chrome_devtools | browser:agent_browser | code-analysis
- screenshots: [before_full_path, before_element_path]
- dom_evidence: 관련 DOM 상태 (evaluate로 수집)
- console_errors: 콘솔 에러 목록 (evaluate로 수집)
