<task type="auto">
  <name>Task 1: classification.md에 sub_type 분류 확장</name>
  <files>plugins/wf/skills/ticket-workflow/references/classification.md</files>
  <action>
    RESEARCH.md Open Question 1의 권장사항에 따라 sub_type 필드를 추가한다.

    1. Classification Output Structure에 `sub_type` 필드 추가:
       ```json
       {
         "ticket_type": "non-ui",
         "sub_type": "logic",
         "confidence": 0.88,
         "ui_score": 0.7,
         "non_ui_score": 5.6,
         "signals": [...],
         "reasoning": "..."
       }
       ```
       - UI 티켓: `sub_type`는 null (Phase 2에서 확장 가능)
       - Non-UI 티켓: `sub_type`는 "logic" | "feature" | "refactoring" | "performance" 중 하나

    2. Non-UI Sub-Type 분류 규칙 섹션 추가:
       - 기존 7차원 신호의 Non-UI 카테고리를 sub_type으로 매핑:
         * `backend` + `data_logic` 신호 우세 -> sub_type: "logic"
         * `infrastructure` + `cli_system` 신호 우세 -> sub_type: "feature" (새 기능/인프라 관련)
         * 리팩토링 관련 키워드(refactor, cleanup, tech debt, code smell, rename, extract, simplify) -> sub_type: "refactoring"
         * 성능 관련 키워드(performance, slow, latency, memory, cache, optimize, bottleneck, profiling) -> sub_type: "performance"
       - sub_type 결정 시 confidence는 ticket_type confidence와 독립 (sub_type은 Non-UI 확정 후 2차 분류)

    3. 성공 기준 2항과의 정합성 확인:
       - "UI/로직/기능/리팩토링/성능 중 하나" 출력은 다음으로 매핑:
         * ticket_type="ui" -> 출력: "UI"
         * ticket_type="non-ui" + sub_type="logic" -> 출력: "로직"
         * ticket_type="non-ui" + sub_type="feature" -> 출력: "기능"
         * ticket_type="non-ui" + sub_type="refactoring" -> 출력: "리팩토링"
         * ticket_type="non-ui" + sub_type="performance" -> 출력: "성능"
       - 이 매핑 테이블을 classification.md에 "## Display Type Mapping" 섹션으로 추가

    4. --type 오버라이드 확장:
       - `--type ui` -> ticket_type="ui", sub_type=null, confidence=1.0
       - `--type non-ui` -> ticket_type="non-ui", sub_type는 자동 분류, confidence=1.0 (ticket_type만)
       - `--type logic` -> ticket_type="non-ui", sub_type="logic", confidence=1.0
       - (같은 패턴으로 feature, refactoring, performance도 지원)
  </action>
  <verify>
    grep -q "sub_type" plugins/wf/skills/ticket-workflow/references/classification.md && echo "SUB_TYPE OK" && grep -q "logic.*feature.*refactoring.*performance" plugins/wf/skills/ticket-workflow/references/classification.md && echo "ALL SUB_TYPES OK" && grep -q "Display Type Mapping" plugins/wf/skills/ticket-workflow/references/classification.md && echo "MAPPING OK"
  </verify>
  <done>classification.md에 sub_type 분류 규칙 + Display Type Mapping + --type 오버라이드 확장이 완성됨</done>
