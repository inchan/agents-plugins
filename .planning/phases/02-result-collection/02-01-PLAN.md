<task type="auto">
  <name>Task 1: 결과 추출 및 정규화 유틸리티 구현</name>
  <files>result-extractor.ts</files>
  <action>
    CLI별 결과에서 핵심 데이터를 추출하고 통일된 형식으로 정규화:

    1. NormalizedResult 타입 정의:
       ```typescript
       interface NormalizedResult {
         success: boolean;
         content: string;          // AI 응답 텍스트
         metadata: {
           cli: string;
           sessionId?: string;
           usage?: { inputTokens: number; outputTokens: number };
           costUsd?: number;
           durationMs: number;
         };
         error?: {
           type: ErrorType;        // 에러 분류
           message: string;
           recoverable: boolean;
         };
       }
       ```

    2. ErrorType enum 정의:
       ```typescript
       type ErrorType =
         | 'CLI_NOT_FOUND'      // CLI가 PATH에 없음
         | 'TIMEOUT'            // 실행 시간 초과
         | 'AUTH_ERROR'         // 인증 실패
         | 'API_ERROR'          // API 호출 실패
         | 'PARSE_ERROR'        // 출력 파싱 실패
         | 'UNKNOWN';           // 기타
       ```

    3. extractResult 함수:
       ```typescript
       function extractResult(runResult: RunResult): NormalizedResult
       ```
       - claude: parsed.result, parsed.session_id, parsed.usage 추출
       - codex: parsed.result 추출
       - echo: parsed.echoed 추출

    4. classifyError 함수:
       ```typescript
       function classifyError(runResult: RunResult): ErrorType
       ```
       - stderr/error 메시지 패턴 분석하여 에러 유형 판별

    **구현 시 주의:**
    - multi-cli-runner.ts의 RunResult 타입 import
    - 각 CLI별 파싱 로직은 별도 함수로 분리
  </action>
  <verify>
    node --import tsx -e "import { extractResult } from './result-extractor.ts'; console.log('import ok')"
  </verify>
  <done>
    - result-extractor.ts 파일 생성됨
    - NormalizedResult, ErrorType 타입 정의됨
    - extractResult, classifyError 함수 구현됨
  </done>
</task>
