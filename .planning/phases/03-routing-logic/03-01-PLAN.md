<tasks>

<task type="auto">
  <name>Task 1: 라우터 타입 및 규칙 정의</name>
  <files>router.ts</files>
  <action>
    AI 라우팅을 위한 타입과 규칙 체계 구현:

    1. RoutingRule 타입 정의:
       ```typescript
       interface RoutingRule {
         name: string;
         priority: number;           // 높을수록 먼저 평가
         match: (request: RoutingRequest) => boolean;
         target: string;             // CLI 이름
         fallback?: string;          // 실패 시 대체 CLI
       }
       ```

    2. RoutingRequest 타입 정의:
       ```typescript
       interface RoutingRequest {
         prompt: string;
         options?: {
           preferSpeed?: boolean;    // 속도 우선
           preferQuality?: boolean;  // 품질 우선
           preferCost?: boolean;     // 비용 우선
           domain?: string;          // 특정 도메인 (code, writing, etc.)
         };
       }
       ```

    3. 기본 라우팅 규칙:
       - 코드 관련 키워드 → codex 우선
       - 일반 대화 → claude 우선
       - 테스트 모드 → echo
  </action>
  <verify>
    node --import tsx -e "import { selectCLI } from './router.ts'; console.log('import ok')"
  </verify>
  <done>
    - router.ts 파일 생성됨
    - RoutingRule, RoutingRequest 타입 정의됨
    - 기본 라우팅 규칙 구현됨
  </done>
</task>

<task type="auto">
  <name>Task 2: 라우터 통합 및 테스트</name>
  <files>router.ts, multi-cli-runner.ts</files>
  <action>
    1. selectCLI 함수 구현:
       ```typescript
       function selectCLI(request: RoutingRequest): string
       ```
       - 규칙을 우선순위 순으로 평가
       - 첫 번째 매칭 규칙의 target 반환
       - 매칭 없으면 기본값(claude) 반환

    2. 라우팅 테스트:
       - 코드 키워드: "코드 작성", "function", "debug" → codex
       - 일반 요청: "안녕", "설명해줘" → claude
       - 테스트: "--test" 옵션 → echo

    3. multi-cli-runner.ts에 --auto 플래그 추가:
       - 라우터가 자동으로 CLI 선택
  </action>
  <verify>

    "
  </verify>
  <done>
    - selectCLI 함수가 요청 특성에 따라 적절한 CLI 반환
    - 코드/일반/테스트 요청 구분 동작
  </done>
</task>
