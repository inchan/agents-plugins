# clarify 테스트 시나리오

## 테스트 목적

PTG Framework와 Information Gain 원칙이 올바르게 적용되는지 검증합니다.

### 평가 기준

| 지표 | 목표 | 설명 |
|------|------|------|
| 질문 정확도 | 질문이 필요한 상황에서만 질문 | True Positive Rate |
| 불필요 질문 회피 | 명확한 상황에서 질문 안 함 | True Negative Rate |
| 컨텍스트 활용 | 가능한 정보로 추론 | 추론 성공률 |
| 질문 품질 | 가장 효과적인 질문 선택 | Information Gain |

---

## 시나리오 1: 질문 필요 - Goal 모호

### 입력
```
프롬프트: "이 코드 정리해줘"
컨텍스트: user.service.ts 파일 열림
```

### PTG 분석
| 요소 | 상태 | 값 |
|------|------|-----|
| Purpose | ❓ 불명확 | 리뷰? 유지보수? |
| Topic | ✅ 명확 | user.service.ts |
| Goal | ❓ 불명확 | 포맷팅? 리팩토링? 삭제? |

### 기대 동작
- **결정**: 질문 필요 (불명확 요소 2개)
- **질문 대상**: Goal (가장 Information Gain 높음)

### 기대 질문
```yaml
question: "'정리'의 범위가 어디까지인가요?"
header: "목표 확인"
options:
  - label: "포맷팅/네이밍"
    description: "들여쓰기, 변수명 정리"
  - label: "구조 개선"
    description: "함수 분리, 모듈화"
  - label: "불필요 코드 제거"
    description: "사용하지 않는 코드 삭제"
  - label: "제가 판단"
    description: "전체적으로 개선"
```

### 성공 조건
- [ ] AskUserQuestion 도구 호출됨
- [ ] Goal에 대한 질문
- [ ] 3개 이상의 구체적 선택지
- [ ] "제가 판단" 옵션 포함


## 평가 매트릭스

| 시나리오 | 질문 필요? | 주요 검증 |
|---------|-----------|----------|
| 1 | ✅ 예 | Goal 모호 → 질문 |
| 2 | ✅ 예 | Topic 참조 불가 → 질문 |
| 3 | ❌ 아니오 | 모두 명확 → 진행 |
| 4 | ❌ 아니오 | 에러 컨텍스트 → 추론 |
| 5 | ❌ 아니오 | 기본값 사용 → 진행 |
| 6 | ✅ 예 | 파괴적 + 범위 불명확 → 질문 |
| 7 | ✅ 예 | 추상적 표현 → 질문 |
| 8 | ❌ 아니오 | 구체적 지시 → 진행 |
| 9 | ✅ 예 | 모순 → 질문 |
| 10 | ❌ 아니오 | 대화 컨텍스트 → 추론 |

### 목표 성공률
- 질문 필요 시나리오 (1,2,6,7,9): 100% 질문 발생
- 질문 불필요 시나리오 (3,4,5,8,10): 0% 질문 발생
