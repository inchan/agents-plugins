   - 모든 시나리오에서 Purpose, Subject, Objective 분석 가능

2. **Information Gain 원칙**: ✅
   - 가장 해소 효과가 큰 모호성 1개만 질문

3. **컨텍스트 활용**: ✅

5. **질문 최소화**: ✅

## 개선 제안

1. **컨텍스트 추론 강화**
   - IDE 상태 (열린 파일, 에러) 적극 활용
   - 예상 질문률 30% → 20%로 감소 가능

2. **도메인별 기본값 확장**
   - React 프로젝트: 컴포넌트 구조 기본값
   - Node.js 프로젝트: 모듈 패턴 기본값

3. **질문 품질 모니터링**
   - 사용자가 "기타"/"직접 설명" 선택 빈도 추적
   - 선택지 개선에 활용
